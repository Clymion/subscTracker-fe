サブスク管理アプリのMVP版の画面遷移図をアーティファクトに格納してあります。
以下は各画面の主要機能です。

1. 認証フロー
   - ログイン画面をエントリーポイントとし、新規登録への導線を用意
   - ログイン後はホーム画面へ遷移

2. メイン機能（Home）
   - サブスクリプション一覧をデフォルト表示
   - 一覧から詳細表示、新規登録、編集が可能
   - ナビゲーションメニューから他の主要機能へアクセス
     - これは他の機能でも共通の機能・パーツ

3. 支払い履歴管理（PaymentHistory）
   - 支払い履歴の一覧表示
   - 支払い詳細の確認と編集
   - 新規支払い記録の追加機能

4. 設定機能（Settings）
   - ユーザープロフィール管理
   - 通貨設定
   - 通知設定
   - ラベル管理（追加・編集）

1~3の画面デザイン（レスポンシブデザイン）は作成済みです。残るは設定画面になりますが、改めて要件から必要な機能と設定画面の構成要素、設定項目を洗い出してください。これが既存の要件と整合性があり必要な項目が揃っているか、UI/UXの観点から改善できるところはないかのレビューもしてください。
その後、設定画面について、どのような画面構成にするのか検討、提案してください。



## 設定機能
設定画面の要件分析と提案を行います。まず、提供された資料から設定関連の要件を整理します。

### 現状の要件と必要な設定項目：

プロフィール設定：
- ユーザー名
- メールアドレス
- パスワード変更
- アカウント削除オプション

通貨設定：
- 基準通貨の選択（ISO 4217準拠）
- 為替レート更新頻度
- 為替レートソースの選択

通知設定：
- 支払い期限通知
- 重要な更新情報の通知
- 通知方法（メール/アプリ内）
- 通知タイミング（何日前か）

ラベル管理：
- ラベル一覧表示
- ラベルの新規作成
- ラベルの編集（名称、色）
- ラベルの削除
